# Prompt สำหรับ Session ถัดไป: แปลศัพท์เฉพาะใน DebriefStage ให้เข้าใจง่ายขึ้น (StageScreen.jsx)

ใช้ prompt นี้เปิด session ใหม่ (หรือวางในข้อความแรก) เพื่อแก้ข้อความส่วน `DebriefStage` ใน
`src/components/StageScreen.jsx` — เป็นแก้ copy/JSX ของ block เดียว ไม่ใช่เปลี่ยน logic คำนวณ

---

## Prompt (คัดลอกทั้งหมดด้านล่างนี้ไปวางใน session ใหม่)

```
งานนี้คือแก้ข้อความในฟังก์ชัน DebriefStage (สเตจ 5 "สรุปเฉพาะตัว") ที่
src/components/StageScreen.jsx บรรทัดประมาณ 193-248 เท่านั้น — เป็นการแก้ copy/JSX
ไม่เปลี่ยนการคำนวณ band.exposure / band.concentration ใดๆ ที่มาจาก engine อ่านโค้ด
ทั้งฟังก์ชันก่อนแก้ เพื่อเห็น state shape จริง (state.band.exposure, state.band.concentration
เป็นตัวเลข 0-1)

## ปัญหาที่ต้องแก้

โค้ดปัจจุบัน (บรรทัด 214-220) แสดงผลแบบนี้:

    <div>
      การเตรียมพอร์ตของคุณ: ความอ่อนไหวต่อ{event.name} <b>{Math.round(band.exposure * 100)}%</b>
      · การกระจุกตัว <b>{Math.round(band.concentration * 100)}%</b>
      {band.concentration > 0.6 && <span> (กระจุกมาก ผลลัพธ์จึงเดาไม่ได้)</span>}
      {band.concentration < 0.25 && <span> (กระจายดี ผลลัพธ์จึงอยู่ในกรอบแคบ)</span>}
    </div>

ปัญหา:
1. "ความอ่อนไหว" (exposure) แสดงเป็น % ล้วนๆ ไม่มีคำแปลความหมายกำกับเลย — เป็นศัพท์ที่เกม
   ประดิษฐ์เอง ผู้เล่น ม.ปลาย ไม่มีทางเดาความหมายจาก "65%" เปล่าๆ ได้
2. "การกระจุกตัว" (concentration) มีคำแปลกำกับแค่ 2 กรณีสุดโต่ง (>60% กับ <25%) ช่วงกลาง
   (25-60%) ไม่มีคำอธิบายอะไรเลย ปล่อยเป็นตัวเลขลอย
3. ทั้งสองอย่างอยู่ใน <div> เดียวกัน ต่อกันด้วย " · " เป็นประโยคยาวเส้นเดียว เสี่ยง wrap
   ล้นบรรทัดแบบไม่เป็นระเบียบ โดยเฉพาะจอมือถือ
4. **บั๊กที่สำคัญที่สุด (พบทีหลัง):** ป้าย "กระจายตัว" กับตัวเลขที่โชว์ (`band.concentration`)
   สวนทางกัน — concentration คือ "ความกระจุกตัว" (ยิ่งเยอะยิ่งแย่) แต่ป้ายบอกว่า "กระจายตัว"
   (ยิ่งเยอะยิ่งดี) พอ concentration = 0 (กระจายดีที่สุด) จะโชว์เป็น "กระจายตัว 0%" ซึ่งอ่าน
   ตรงตัวแล้วหมายถึง "ไม่กระจายเลย" — ตรงข้ามกับความจริงโดยสิ้นเชิง ต้องแก้ที่ตัวเลขให้ตรง
   กับป้าย ไม่ใช่แค่เติมสี/คำอธิบาย (ดูข้อ 2 ด้านล่าง)

## การแก้ไข

### 1. แยกเป็น 2 บรรทัด/บล็อกอิสระ คนละ <div> (กันล้น ไม่ให้ประโยคยาวมาต่อกัน)
บรรทัดละ 1 สถิติ ประโยคแปลความหมายอยู่ในบรรทัดเดียวกับตัวเลขของมันเอง ไม่ให้ยาวข้าม
ไปกระทบเลย์เอาต์ของอีกสถิติหนึ่ง

### 2. เติมคำแปลความหมายให้ครบทุกช่วง ไม่ใช่แค่กรณีสุดโต่ง

**Exposure** (`band.exposure`, 0-1) — โครงประโยค: "อ่อนไหวต่อ{event.name} {n}% — {คำแปล}"
| ช่วง | คำแปล |
|---|---|
| >= 0.6 | สินทรัพย์ส่วนใหญ่ในพอร์ตอ่อนไหวกับเหตุการณ์แบบนี้ |
| 0.35 - 0.6 | พอร์ตโดนเหตุการณ์นี้กระทบปานกลาง |
| < 0.35 | สินทรัพย์ส่วนใหญ่ในพอร์ตไม่ค่อยเกี่ยวกับเหตุการณ์นี้ |

**Concentration → แสดงเป็น "การกระจายตัว" ที่พลิกค่าแล้ว (สำคัญ):** ห้ามโชว์ `band.concentration`
ดิบๆ ใต้ป้าย "กระจายตัว" เพราะสองอย่างนี้ความหมายสวนทางกัน (ดูปัญหาข้อ 4 ด้านบน) ให้คำนวณ
ค่าที่โชว์ผู้เล่นเป็น **`diversification = 1 - band.concentration`** แทน (แปลงตอนแสดงผล
เท่านั้น ไม่แตะค่า `band.concentration` ที่ engine ใช้คำนวณจริงที่อื่น) โครงประโยค:
"กระจายตัว {Math.round(diversification * 100)}% — {คำแปล}" — ตอนนี้ยิ่งเลขเยอะยิ่งดีจริง
ตรงกับป้ายแล้ว (concentration 0 → diversification 100% → "กระจายตัว 100%" อ่านแล้วเข้าใจถูก
ทันทีว่าคือกรณีดีที่สุด ไม่ต้องพึ่งสีตีความอีก)

| diversification (= 1 - concentration) | เท่ากับ concentration ดิบ | คำแปล |
|---|---|---|
| >= 0.75 | < 0.25 | กระจายดี ผลลัพธ์จึงอยู่ในกรอบแคบ |
| 0.4 - 0.75 | 0.25 - 0.6 | กระจายพอสมควร ผลลัพธ์จึงไม่สุดโต่งไปทางใดทางหนึ่ง |
| < 0.4 | > 0.6 | กระจุกมาก ผลลัพธ์จึงเดาไม่ได้ |

### ตัวอย่างผลลัพธ์ที่ต้องการ (แทนที่บรรทัด 214-220 เดิม)

    const diversification = 1 - band.concentration
    <div>อ่อนไหวต่อ{event.name} <b>{Math.round(band.exposure * 100)}%</b> — {exposureNote}</div>
    <div className="mt-1">กระจายตัว <b>{Math.round(diversification * 100)}%</b> — {diversificationNote}</div>

โดย exposureNote / diversificationNote เป็นฟังก์ชัน helper เล็กๆ (เขียนไว้เหนือ component หรือ
ในไฟล์เดียวกัน คล้ายรูปแบบ luckLabel()/prepLabel() ที่มีอยู่แล้วใน src/game/engine/report.js)
รับค่าตัวเลข 0-1 คืนสตริงคำแปลตามตารางข้างบน (diversificationNote รับค่า `diversification`
ที่แปลงแล้ว ไม่ใช่ concentration ดิบ)

### 3. ใส่กรอบ + สีไฮไลต์ตามโทน "ข่าวดี/ข่าวร้าย/กลาง" ให้ทั้ง 3 บรรทัด (exposure,
diversification, และบรรทัด luck ที่มีอยู่แล้วบรรทัด 222-225) — สีเป็นตัวเสริมความชัดเจน
ไม่ใช่ตัวแก้หลัก (ตัวแก้หลักคือพลิกเลขให้ตรงกับป้ายตามข้อ 2 ด้านบนแล้ว) ผู้เล่นกวาดตาจับสีได้
เร็วกว่าอ่านตัวเลขอยู่ดี จึงยังควรใส่ไว้เป็นสัญญาณซ้อนอีกชั้น

ให้แต่ละบรรทัดเป็นกล่อง `pixel-frame` มีขอบ+พื้นหลังตามโทน ใช้ pattern สีเดียวกับที่ใช้อยู่แล้ว
ในไฟล์นี้และ ReportScreen.jsx (เช่น `border-rose-500/50 bg-rose-950/40 text-rose-300` สำหรับ
โทนแย่, `border-emerald-500/50 bg-emerald-950/40 text-emerald-300` สำหรับโทนดี,
`border-slate-700 bg-slate-900/70 text-white/70` สำหรับกลาง) — เขียนเป็น object/ฟังก์ชัน
map โทน → className เดียว ใช้ซ้ำได้ทั้ง 3 บรรทัด เช่น:

    const TONE_CLS = {
      bad: 'border-rose-500/50 bg-rose-950/40 text-rose-300',
      good: 'border-emerald-500/50 bg-emerald-950/40 text-emerald-300',
      neutral: 'border-slate-700 bg-slate-900/70 text-white/70',
    }

เกณฑ์กำหนดโทนต่อบรรทัด (ใช้เกณฑ์เดิมที่ผ่านการออกแบบไปแล้ว ไม่ต้องคิดใหม่ — ระวัง
diversification ใช้เกณฑ์ตรงข้ามกับ concentration ดิบ เพราะเป็นค่าที่พลิกแล้ว):
- **exposure**: >= 0.6 → bad, < 0.35 → good, ระหว่างนั้น → neutral
- **diversification** (ค่าที่พลิกแล้ว = 1 - concentration): >= 0.75 → good, < 0.4 → bad,
  ระหว่างนั้น → neutral (เทียบเท่า concentration > 0.6 → bad, < 0.25 → good แบบเดิม
  แค่คำนวณกลับด้าน อย่าใช้เกณฑ์ 0.6/0.25 ตรงๆ กับค่า diversification เพราะจะกลับด้านผิด)
- **luck**: ใช้ threshold ที่มีอยู่แล้วในโค้ดบรรทัด 224 ตรงๆ (luckPct >= 55 → good,
  luckPct >= 45 → neutral, ต่ำกว่านั้น → bad) — อย่าสร้างเกณฑ์ใหม่ซ้ำซ้อนกับที่มีอยู่

แต่ละกล่องห่อด้วย `<div className={`pixel-frame border p-1.5 sm:p-2 ${TONE_CLS[tone]}`}>` ครอบ
ทั้งบรรทัดตัวเลข+คำแปล (ไม่ใช่แค่ห่อคำแปลอย่างเดียว) เว้นระยะห่างระหว่างกล่องด้วย
`mt-1.5`/`gap-1.5` ให้เข้าชุดกับ spacing ที่ใช้อยู่ในไฟล์นี้แล้ว

## ข้อควรระวัง

- ห้ามเปลี่ยนค่า band.exposure / band.concentration หรือ threshold ที่ engine ใช้คำนวณจริง
  (เช่น BALANCE.bandWidthPerConcentration ใน balance.js) — นี่คือการแก้ข้อความแสดงผลเท่านั้น
  `diversification = 1 - band.concentration` เป็นค่าที่คำนวณเฉพาะตอนแสดงผลใน component นี้
  เท่านั้น ไม่ส่งค่านี้กลับเข้า engine หรือ state ที่อื่น
- เกณฑ์ในตารางข้างบนเป็นเกณฑ์ "คำอธิบาย/สี" ใช้ตามตารางตรงๆ — ระวังอย่าสับสนว่าเกณฑ์ไหน
  ใช้กับ concentration ดิบ (ของ engine) เกณฑ์ไหนใช้กับ diversification ที่พลิกแล้ว
  (ของการแสดงผล) ทั้งสองพูดถึงข้อเท็จจริงเดียวกันแค่กลับด้านตัวเลข
- ส่วนที่เหลือของ DebriefStage (hero/villain tool, ค่าธรรมเนียม, black swan, บทเรียนปิดท้าย)
  ไม่ต้องแตะ — คงไว้เหมือนเดิมทั้งหมด
- ทดสอบใน dev server จริง (npm run dev) เล่นจนถึงสเตจ 5 ของบทใดก็ได้ เช็คว่าข้อความไม่ล้น
  บรรทัดที่จอมือถือ (resize แคบดู)
```
