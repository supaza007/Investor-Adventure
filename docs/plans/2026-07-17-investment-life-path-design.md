**แทนที่:** ระบบเกมการ์ดต่อสู้แบบ Slay the Spire เดิม (deck-builder ตีศัตรู HP)
**วันที่ออกแบบ:** 2026-07-17

## 1. ปัญหาที่พบจากเกมเดิม (ทำไมต้องออกแบบใหม่)

จาก playtest จริง:
1. **เกมยืดน่าเบื่อ** — สาเหตุทางคณิตศาสตร์: แต่ละคลาสมีการ์ดไม่ซ้ำแค่ 5 ใบ ผู้เล่นเห็นครบภายใน 2-3 เทิร์นแรก พื้นที่การตัดสินใจ (decision space) ยุบเหลือ 0 หลังจากนั้น แต่การต่อสู้ (ศัตรู HP 39-54 ปะทะดาเมจการ์ด 4-12) ยังลากยาวต่ออีกหลายเทิร์น — ผู้เล่นแค่กดปุ่มเดิมซ้ำรอเลขคำนวณเสร็จ
2. **ผู้เล่นเมินความรู้ สนใจแต่ค่าโจมตี/ป้องกัน** — เพราะความรู้การลงทุนอยู่ใน `lesson` popup ซึ่งเป็น**ข้อความอธิบายกลไก** ไม่ใช่**กลไกเอง** เกมให้คะแนนแค่ค่าโจมตี/ป้องกัน ผู้เล่นจึง optimize สิ่งที่เกมนับ — ซึ่งเป็นพฤติกรรมที่ถูกต้องตามกติกาที่ออกแบบไว้ ไม่ใช่ความผิดของผู้เล่น

**ข้อบกพร่องเชิงโครงสร้างที่เป็นต้นตอ:**
- ผู้เล่น = สินทรัพย์เดียวตลอดเกม (สอนตรงข้ามกับการกระจายความเสี่ยง)
- การ์ด = ใช้แล้วทิ้ง (สอนพฤติกรรมเก็งกำไรระยะสั้น ไม่ใช่การถือครองแบบนักลงทุน)
- เงินเฟ้อ/วิกฤต = ศัตรูมี HP ให้ฆ่าตาย (ทั้งที่ในโลกจริงไม่มีใครฆ่าเงินเฟ้อได้ มันคือสภาพอากาศที่ต้องอยู่รอด ไม่ใช่บอสให้เอาชนะ)
- Portfolio = หลอดเลือดที่ป้องกันไม่ให้ถึง 0 (ทั้งที่ควรเป็นคะแนนที่ทำให้โต)
- ลำดับเกม = เห็นเหตุการณ์ก่อนแล้วค่อยเลือกเครื่องมือ (กลายเป็นข้อสอบท่องเฉลย ไม่ใช่การตัดสินใจภายใต้ความไม่แน่นอนแบบการลงทุนจริง)

## 2. วิสัยทัศน์หลักและเสาหลัก 4 ข้อ

**คำกริยาหลักของเกมเปลี่ยนจาก "โจมตี" เป็น "จัดสรร"**

1. **ถือ ไม่ใช่ตี** — ตำแหน่งการลงทุนที่เลือกอยู่บนโต๊ะต่อเนื่อง รับผลจากหลายเหตุการณ์ ไม่ใช่ใช้ครั้งเดียวทิ้ง
2. **ตัดสินใจก่อนรู้ผล** — จัดพอร์ตก่อนเห็นรายละเอียดเหตุการณ์ เวลาเดินหน้าอย่างเดียว แก้ย้อนหลังไม่ได้
3. **ไม่มีเฉลยสมบูรณ์แบบ** — ทุกเครื่องมือมีจุดแข็ง-จุดอ่อนจริง กระจุกตัวต้องเจ็บจริงเมื่อเจอเหตุการณ์ที่ตรงจุดอ่อน (เรียนรู้จากผลลัพธ์ ไม่ใช่จาก popup)
4. **โตหรือฟุบ ไม่ใช่ตายหรือรอด** — Portfolio Value คือคะแนนสะสม ไม่ใช่ HP; แพ้ได้เป็นระดับ ไม่ใช่ 0/1

**สิ่งที่เกมนี้ไม่ใช่:** ไม่มีศัตรูให้ตีตาย, ไม่มีการ์ดใช้แล้วทิ้ง, ไม่มีเฉลยตายตัวให้ท่องจำ, ตัดสินใจถูกไม่การันตีว่าชนะ

## 3. โครงสร้างเกม: เส้นทางชีวิต 6 บท (อายุ 20→60)

เล่นจบ 1 รอบเป้าหมาย **10-15 นาที** (เหมาะเล่นในคาบเรียนเดียว เล่นซ้ำได้ด้วยสไตล์อื่น)

| บท | ช่วงอายุ | ธีม |
|---|---|---|
| 1 | 20-29 | เริ่มทำงาน ทุนน้อย เสี่ยงได้มาก |
| 2 | 30-39 | ภาระเพิ่ม (ครอบครัว/บ้าน) |
| 3 | 40-49 | รายได้พีค แต่เจอวิกฤตใหญ่ที่สุด |
| 4 | 50-59 | ต้องเริ่มลดความเสี่ยงก่อนเกษียณ |
| 5 | 60 | เกษียณ — รายงานผลสรุป |

แต่ละบท (ยกเว้นบทสุดท้าย) มีโครง 3 ช่วง:

1. **ช่วงจัดพอร์ต (Allocation)** — ทุน/พลังงานจำกัด สร้าง/ปรับตำแหน่งถือครอง **ก่อนรู้รายละเอียดเหตุการณ์บทนี้** (รู้แค่บริบทกว้าง เช่น "อายุ 35 แล้ว")
2. **ช่วงเหตุการณ์คลี่ตัว (Unfolding)** — 5 การตัดสินใจต่อบท:
   - **สเตจ 1:** สัญญาณเตือนคลุมเครือ — ปรับพอร์ตเล็กน้อยได้ ทุนจำกัด
   - **สเตจ 2:** เผยประเภทเหตุการณ์บางส่วน — ตัดสินใจอีกครั้งด้วยข้อมูลมากขึ้น
   - **สเตจ 3 (จุดพีค):** แรงกระแทกจริงลงพอร์ต คำนวณจาก exposure tag overlap + ความสุ่มขอบเขตแคบ
   - **สเตจ 4 (จุดพฤติกรรม):** ถือต่อ / ตัดขาดทุน / ซื้อเพิ่มตอนราคาถูก — สอน panic-selling ผ่านการกระทำ
   - **สเตจ 5 (สรุปเฉพาะตัว):** อธิบายตัวเลขจริงว่าเหตุใดผลลัพธ์จึงเป็นแบบนี้ (เช่น "ตราสารหนี้ดูดแรงกระแทกไป 60%")
3. **ช่วงสรุปผล (Snapshot)** — Portfolio Value อัปเดตถาวร ตำแหน่งที่รอดต่อไปบทถัดไป (ไม่รีเซ็ต) มีการทบต้นเงียบๆ ระหว่างบท (ให้รางวัลการถือ ไม่ใช่รีบใช้การ์ดให้หมด)

## 4. Encounter Engine: ระบบ Exposure Tag (แทน attack/block/debuff เดิม)

ทุกเครื่องมือมี **Exposure Tag** (ความอ่อนไหว) ทุกเหตุการณ์มี **Event Tag** เช่นกัน ผลลัพธ์ = การทับซ้อนของสองแท็กนี้ — พอร์ตกระจุกตัวโดนตรงจุดจะเจ็บหนักโดยอัตโนมัติ นี่คือกลไกที่สอนการกระจายความเสี่ยงผ่านผลลัพธ์ ไม่ใช่ผ่านข้อความ

**Tag ความอ่อนไหว 4 แบบ:**
| Tag | เหตุการณ์ที่เกี่ยวข้อง (จาก enemies.json เดิม) |
|---|---|
| 📈 เติบโต | สงคราม, โรคระบาด, ภัยธรรมชาติ |
| 💸 เงินเฟ้อ/ค่าเงิน | เงินเฟ้อ, ต้มยำกุ้ง, จีน (reserve_boss) |
| 💧 สภาพคล่อง/หนี้ | ฟองสบู่, กิเลส (greed), ทรัมป์ (tariff_boss) |
| 😨 จิตวิทยาตลาด | ความกลัว, การเมือง, Scammer |

**เนื้อหาเหตุการณ์ทั้ง 12 ตัวใน `enemies.json` เดิม ใช้ต่อได้เลย** แค่เปลี่ยนจาก "ศัตรูมี HP" เป็น "เหตุการณ์มี tag" — ไม่ต้องคิดเนื้อหาใหม่ตั้งแต่ศูนย์

## 5. คลังเครื่องมือการเงิน 6 ชนิด (ใช้ร่วมกันทุกสไตล์)

**สำคัญ:** แกน "เครื่องมือการเงิน" แยกจากแกน "สไตล์นักลงทุน" — ทุกสไตล์ซื้อเครื่องมือไหนก็ได้ ต่างกันแค่ "วิธีเล่นกับมัน" (ดูข้อ 6) นี่คือจุดที่ต่างจากเกมเดิมที่ผูกคลาส = สินทรัพย์เดียวตลอดเกม

| เครื่องมือ | ความอ่อนไหวหลัก | บทบาท |
|---|---|---|
| ตราสารหนี้ | ต่ำทุก tag | ปลอดภัยสุด (safe-haven, แรงหน่วงสูงสุด) |
| ESG (หุ้นยั่งยืน) | เติบโต/จิตวิทยา ต่ำ | โตช้าแต่มั่นคง |
| กองทุนรวม | กลางทุก tag | มีกระจายความเสี่ยงในตัว (ลดโทษการกระจุกอัตโนมัติ) |
| หุ้น | เติบโต สูง | เสี่ยงสูง-ผลตอบแทนสูง |
| อนุพันธ์ | สภาพคล่อง สูงสุด | เลเวอเรจ ขยายทั้งกำไร/ขาดทุน เสี่ยงล้มละลายจริง |
| คริปโตเคอเรนซี | จิตวิทยา+เติบโต สูงสุด | ผันผวนสุดในเกม ไม่มีแรงหน่วง เสี่ยงล้มละลายจริง จับคู่กับเหตุการณ์ Scammer ได้ตรงประเด็น |

คลังนี้ต่อยอดจาก `classes.json` เดิม (หุ้น/ตราสารหนี้/กองทุนรวม/อนุพันธ์/ESG) แค่เพิ่มคริปโต และเปลี่ยนจาก "การ์ดเฉพาะคลาส" เป็น "คลังกลาง"

## 6. สไตล์นักลงทุน 4 แบบ (แทนคลาสตัวละครเดิม)

หัวใจ: แต่ละสไตล์บังคับให้เล่นกับ Encounter Engine ต่างกันจริง ไม่ใช่แค่เปลี่ยนเลข/สกิน

| สไตล์ | กลไกเฉพาะ | บทเรียนที่สอน |
|---|---|---|
| 🗡️ เทรดเดอร์ | ปรับพอร์ตได้ทุกสเตจ แต่มีค่าธรรมเนียมทุกครั้ง ตำแหน่งไม่ทบต้นเองถ้าไม่จัดการ | Overtrading กัดกินผลตอบแทน |
| 🛡️ ระยะยาว | ปรับพอร์ตได้แค่ 1 ครั้ง/บท ทบต้นแรงสุด แรงกระแทกถูกหน่วงมากสุด | เวลาในตลาดสำคัญกว่าจับจังหวะตลาด |
| ⚖️ ระยะกลาง | ปรับได้แค่จุดกึ่งกลาง (สเตจ 2-3) ทุกอย่างกลางๆ | คลาสเริ่มต้น/สอนระบบ |
| 🌱 VI (Value Investor) | ได้โบนัสก้อนใหญ่เมื่อ "ซื้อเพิ่ม" ที่จุดตัดสินใจพฤติกรรม (สเตจ 4) ตอนราคาร่วง | กลัวตอนคนอื่นโลภ โลภตอนคนอื่นกลัว |

## 7. ระบบแฟร์เนส: "ตัดสินใจถูกแล้วยังแพ้ได้" โดยไม่รู้สึกโกง

ทุกจุดพีค (สเตจ 3) เกมสร้าง **outcome band** จากคุณภาพการเตรียมพอร์ต (กระจายดี = ช่วงแคบเลื่อนบวก, กระจุก = ช่วงกว้างเลื่อนลบ) แล้วสุ่มผลจริงจากในช่วงนั้น ตอนสรุปผล (สเตจ 5) โชว์ทั้งคู่เทียบกัน: *"การเตรียมของคุณอยู่กลุ่มดีที่สุด 20% แต่ผลที่ออกมาคือกรณีโชคร้ายในกลุ่มนั้น"* — ผู้เล่นเห็นว่าตัวเองตัดสินใจถูก แค่จับสลากไม่ดี ไม่ใช่เกมโกง

**Black Swan (หายาก ~10-15% ของจุดพีค):** เหตุการณ์ไม่มีแท็กป้องกันได้เลย พร้อมข้อความชัดว่า "แม้นักลงทุนเก่งที่สุดก็เลี่ยงไม่ได้" ต้องหายากพอไม่ให้เป็นข้ออ้างละเลยการกระจายความเสี่ยง

**ไม่มี Game Over ทันทีจากบทเดียว** — บทหนึ่งพังไม่จบเกม ยกเว้นล้มละลายจริง (มูลค่าติดลบ/แตะ 0 จากเลเวอเรจเกินตัว — อนุพันธ์/คริปโต)

**จบเกมที่อายุ 60 = "รายงานผลเกษียณ"** ไม่ใช่ win/lose แต่เป็นสเปกตรัม เทียบกับเกณฑ์อ้างอิง (เช่น "ถ้าถือกองทุนดัชนีเฉยๆ ตลอด 40 ปีจะได้เท่าไหร่") พร้อมระดับผล (เกษียณสบาย/พอใช้/ล้มละลาย/FIRE ก่อนกำหนด) และสรุปว่าบทไหนช่วย/ทำร้ายพอร์ตและทำไม

## 8. สิ่งที่ยังไม่ได้ตัดสินใจ (ต้องคุยต่อก่อน implement)

- หน้าตา UI คร่าวๆ ของช่วง Allocation และ Unfolding (ยังไม่ได้ออกแบบ)
- รายละเอียดตัวเลขจริงของ outcome band / สูตรคำนวณ tag overlap
- กลไกจุดตัดสินใจพฤติกรรม (สเตจ 4) แบบละเอียด — ตัวเลือก "ถือ/ตัดขาดทุน/ซื้อเพิ่ม" ให้ผลต่างกันยังไงเชิงตัวเลข
- Scammer event อาจต้องมีกลไกพิเศษเฉพาะ (ไม่ใช่แค่ tag matching) เพราะเป็นกรณีที่ตัดสินใจของผู้เล่นเองมีผลโดยตรง (เช่น มีตัวเลือก "ข้อเสนอดีเกินจริง" ให้ตกหลุมพรางได้)
- ทุนเริ่มต้น/พลังงานต่อบท ตัวเลขจริง
- เนื้อหาเชิงเรื่องราว/บทพูดของแต่ละสเตจ

## 9. หมายเหตุ: นี่คือการรีดีไซน์ ไม่ใช่การปรับแต่ง

โครงสร้างนี้เปลี่ยนแกนเกมทั้งหมด (ต่อสู้ → จัดพอร์ต) โค้ดเดิมที่มีอยู่ (`reducer.js` 629 บรรทัด, `EnemyPanel.jsx`, `CardConfirm.jsx` ฯลฯ ทั้งหมดเป็นกลไกต่อสู้แบบเดิม) **น่าจะนำมาใช้ซ้ำได้น้อยมาก** ควรวางแผนเป็นการเขียนระบบเกมใหม่เกือบทั้งหมด ไม่ใช่แก้ไขของเดิม ส่วนที่ยังใช้ได้แน่ๆ คือ `enemies.json` และ `classes.json` เดิม (ใช้เป็นข้อมูลตั้งต้น ปรับ schema เล็กน้อย) และ asset ภาพ/เสียงต่างๆ
