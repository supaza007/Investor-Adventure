ใช้ prompt นี้เปิด session ใหม่ (หรือวางในข้อความแรก) — งานนี้ผ่านการออกแบบ/ทำ mockup ร่วมกับ
ผู้ใช้มาแล้ว (ไม่ต้อง re-design แนวคิดใหม่) มี 2 ฟีเจอร์รวมกันเพราะฟีเจอร์ 2 (ปุ่มดูเหตุการณ์

ที่คลิกจุดที่ผ่านมาแล้วได้ ก็คือ "ดูเหตุการณ์ย้อนหลัง" ในตัว ไม่ต้องสร้างปุ่มแยกอีกอัน

```
งานนี้มี 2 ฟีเจอร์ ทำใน src/components/AllocationScreen.jsx และ src/components/StageScreen.jsx
เป็นหลัก อ่านทั้งสองไฟล์ก่อน (180 และ 334 บรรทัด) รวมถึง src/game/engine/data/tools.js

## ฟีเจอร์ 1: การ์ดเครื่องมือแบบย่อ (มือถือ) + กดดูรายละเอียดเต็ม ใน AllocationScreen.jsx

### ปัญหา
`ToolRow` (บรรทัด 11-47) ตอนนี้มี 4 บรรทัดข้อความต่อใบ (ชื่อ, tagline ยาว, "โต X× / 10 ปี",
"จุดอ่อน [tag ที่แย่สุด 1 อัน] {riskDots}") ในการ์ดกว้างแค่ ~177px บนมือถือ (grid 2 คอลัมน์)
— หนาแน่นเกินไป ("ตัวหนังสือไม่ต้องเยอะ" ตามที่ผู้ใช้ต้องการ)

ปัญหาที่ซ่อนอยู่: **"จุดอ่อน [tag]" เทียบกันข้ามการ์ดไม่ได้** เพราะแต่ละเครื่องมือโชว์คนละ tag
(ตราสารหนี้ = "เงินเฟ้อ/ค่าเงิน", หุ้น = "เติบโต") คนละแกนกัน ต่อให้อ่านครบ 6 ใบก็ไม่รู้ว่า
ใบไหนเสี่ยงกว่าใบไหน "โดยรวม" — ไม่ใช่แค่ข้อความเยอะ แต่ข้อมูลที่โชว์อยู่ก็เทียบกันจริงไม่ได้

นอกจากนี้ `growthVol` (ความผันผวน) กับ `lesson` (คำอธิบายเชิงความรู้) มีอยู่ในข้อมูลจริง
(`src/game/engine/data/tools.js`) แต่ไม่เคยถูกโชว์ที่หน้านี้เลย

### การแก้ไข — สรุปมาจาก mockup ที่ทำร่วมกับผู้ใช้แล้ว (ยืนยันแบบสุดท้ายแล้ว ไม่ต้องเสนอ
ทางเลือกใหม่)

**1. ย่อการ์ดหน้าเริ่มต้นเหลือ 3 บรรทัด** (คงเลย์เอาต์ grid 2 คอลัมน์เดิม `grid-cols-2` ไว้
ตามเดิม — ยืนยันแล้วว่าเห็นครบ 6 ใบไม่ต้องเลื่อนดีกว่า):
- บรรทัด 1: ชื่อเครื่องมือ (เหมือนเดิม)
- บรรทัด 2: % กับจำนวนเงิน **อยู่บรรทัดเดียวกัน** จัดชิดซ้าย-ขวา (`justify-between`) — % ตัว
  ใหญ่ตัวหนา จำนวนเงินตัวเล็กกว่าสีจาง
- บรรทัด 3: ปุ่ม −/+ (เหมือนเดิม)
- บรรทัด 4: **"เสี่ยง {dots}"** ระดับความเสี่ยงโดยรวม **แทนที่** "จุดอ่อน [tag]" เดิมทั้งหมด
  — คำนวณจาก `growthVol` (มาตราเดียวกันทุกเครื่องมือ จึงเทียบข้ามการ์ดได้จริง ต่างจาก
  exposure ต่อ tag ที่เทียบกันไม่ได้):
  | ช่วง growthVol | จำนวนจุด |
  |---|---|
  | < 0.15 | ● (1 จุด) |
  | 0.15 - 0.35 | ●● (2 จุด) |
  | 0.35 - 0.6 | ●●● (3 จุด) |
  | 0.6 - 1.0 | ●●●● (4 จุด) |
  | > 1.0 | ●●●●● (5 จุด) |
  เขียนเป็นฟังก์ชัน helper แยก (เช่น `volDots(growthVol)`) ไม่ปนกับ `riskDots(v)` เดิมที่ใช้
  กับ exposure 0-1 เพราะ scale คนละแบบ

**tagline และ "โต X× / 10 ปี" ถูกตัดออกจากหน้าการ์ดทั้งคู่** — ย้ายไปอยู่ใน modal
รายละเอียดแทน (ดูข้อ 2) ไม่ได้หายไปจากเกม แค่ไม่ได้อยู่หน้าเริ่มต้นแล้ว

**2. เพิ่ม modal รายละเอียด** เปิดเมื่อกดที่ตัวการ์ด `ToolRow` (ยกเว้นปุ่ม +/− ที่ต้องกดได้
ตามปกติ ไม่ให้เปิด modal) — ใช้ pattern overlay เดียวกับ `ChapterIntroModal` ที่มีอยู่แล้วใน
ไฟล์นี้ (บรรทัด 51-81, `fixed inset-0 z-50 bg-black/85`) ไม่ต้องคิด pattern ใหม่ ใส่ไอคอน
เล็กๆ (เช่น "ⓘ") มุมการ์ดเป็นตัวใบ้ว่ากดดูเพิ่มได้ — ผู้เล่นที่ไม่อยากอ่านลึกไม่ต้องกดเลยก็เล่น
เกมได้ปกติ

Modal แสดง:
1. ชื่อ + tagline (ย้ายมาจากหน้าการ์ด)
2. **"โต {growthMult.toFixed(2)}× / 10 ปี"** (ย้ายมาจากหน้าการ์ด)
3. **ความอ่อนไหวครบทั้ง 4 tag** (ไม่ใช่แค่ตัวแย่สุด) — วน `Object.entries(tool.exposure)`
   ทั้งหมด แสดงเป็นแถวๆ ใช้ `TAG_LABELS[tag]` + `riskDots(value)` ที่มีอยู่แล้วในไฟล์นี้
   (บรรทัด 9) ซ้ำได้เลย ไม่ต้องเขียนใหม่ — นี่คือจุดที่ "จุดอ่อนจริงครบด้าน" ที่หายไปจากหน้า
   การ์ดกลับมาอยู่ครบ
4. **ความผันผวน (growthVol)** แปลเป็นคำ (ใช้ตารางเดียวกับที่กำหนด `volDots` ข้อ 1 แค่โชว์
   คำแทนจุด): < 0.15 แทบไม่ผันผวน, 0.15-0.35 ผันผวนต่ำ, 0.35-0.6 ผันผวนปานกลาง,
   0.6-1.0 ผันผวนสูง, > 1.0 ผันผวนสุดขั้ว
5. `tool.lesson` (มีอยู่แล้วในข้อมูล ไม่เคยถูกโชว์ในหน้านี้มาก่อน) — ข้อความล้วน ไม่ต้องแก้

## ฟีเจอร์ 2: Life Timeline (แทนที่ "ปุ่มดูเหตุการณ์ย้อนหลัง" เดิม)

### สร้าง component ใหม่ src/components/LifeTimeline.jsx
รับ props: `chapters` (จาก `BALANCE.chapters`), `currentChapterN` (เลขบทปัจจุบัน),
`history` (จาก `state.history` — array เดียวกับที่ ReportScreen.jsx ใช้สร้างรายงานท้ายเกม)

แสดงเป็นแถบแนวนอน มีเส้นเชื่อม + จุดวงกลมทั้งหมด `BALANCE.chapters.length + 1` จุด
(4 บทจริง + 1 จุดเกษียณอายุ `BALANCE.retireAge`) แต่ละจุดมี 3 สถานะ:
- **ผ่านมาแล้ว** (n < currentChapterN): วงกลมเขียว ไอคอนถูก **กดได้**
- **ปัจจุบัน** (n === currentChapterN): วงกลมสีเน้น มี emoji ตัวละคร (เช่น 🧑) อยู่ข้างใน
- **อนาคต** (n > currentChapterN หรือจุดเกษียณ): วงกลมจาง ไอคอนแม่กุญแจ **กดไม่ได้**
  (ห้ามให้ดูเหตุการณ์ล่วงหน้าได้เด็ดขาด — ขัดกับดีไซน์หลัก "ตัดสินใจก่อนรู้ผล")

กดจุด "ผ่านมาแล้ว" เปิดกล่องสรุปเล็กๆ ด้านล่างแถบ (ไม่ต้องเป็น modal เต็มจอ แค่ expand
inline พอ) แสดง: ชื่อเหตุการณ์ของบทนั้น (`history[i].eventName`), การเตรียมพอร์ต+ดวง
(เอาค่า `exposure`/`concentration`/`percentile` ของบทนั้นจาก history ไปเข้า `prepLabel()`/
`luckLabel()` จาก src/game/engine/report.js — **ใช้ฟังก์ชันเดิมซ้ำ ไม่เขียนใหม่**), และ
% เปลี่ยนแปลงมูลค่าของบทนั้น (คำนวณแบบเดียวกับที่ `buildReport()` ทำกับ `changePct`)

### จุดติดตั้ง
Import `LifeTimeline` เข้า **ทั้ง** `AllocationScreen.jsx` (วางเป็นแถวใหม่เหนือ `<header>`
เดิมที่บรรทัด 126) **และ** `StageScreen.jsx` (วางเป็นแถวใหม่เหนือ `<header>` เดิมที่บรรทัด
290) — ไม่ต้องซ้ำโค้ด markup ในสองไฟล์ ให้ทั้งคู่เรียก component เดียวกัน แถวเดิม (ป้าย
"บทที่ N · อายุ..." + StageTrack) อยู่ต่อด้านล่างเหมือนเดิมทุกอย่าง ไม่ต้องแก้

## ข้อควรระวัง

- ฟีเจอร์ 1 และ 2 เป็นคนละส่วนของโค้ด ทำแยกกันได้ ไม่ต้องพึ่งกัน
- เกมนี้ mobile-first จอแนวตั้งเป็นหลัก (ฟอนต์เล็กสุดในโค้ดคือ text-[8px] เพราะพื้นที่จำกัด
  จริงๆ) — LifeTimeline ต้องกระทัดรัดที่สุด (จุดวงกลมเล็ก ~20-24px ไม่ใหญ่กว่านี้) ทดสอบบน
  breakpoint มือถือจริงว่าไม่ดันเนื้อหาเกมหลักลงจนต้อง scroll เพิ่ม
- ห้ามให้จุดอนาคตกดดูอะไรได้เลย แม้แต่ hover/title ก็ไม่ควรใบ้เนื้อหา — จุดนี้เป็นกติกาหลัก
  ของเกมที่ผ่านการออกแบบมาแล้ว (docs/plans/2026-07-17-investment-life-path-design.md ข้อ 2)
- คงสไตล์ pixel-art เดิมของเกมทั้งหมด (pixel-frame, pixel-chip, สีจาก ToolTheme.js/colorOf())
  ไม่ใช้สไตล์ flat ทั่วไป
- หลัง implement เสร็จ เปิด dev server จริง (npm run dev) เล่นผ่านอย่างน้อย 2 บทเต็ม เช็คว่า:
  1. การ์ดหน้าเริ่มต้นเหลือ 3-4 บรรทัดตามสเปก ("เสี่ยง {dots}" แทน "จุดอ่อน [tag]" เดิม)
     grid ยังคง 2 คอลัมน์ เห็นครบ 6 ใบไม่ต้องเลื่อนบนจอมือถือจริง
  2. กด ToolRow เปิด modal รายละเอียดถูกต้อง (มี tagline, โต X×, exposure ครบ 4 tag,
     growthVol แปลเป็นคำ, lesson) กด +/− ไม่เผลอเปิด modal
  3. LifeTimeline โชว์ถูกทั้งใน AllocationScreen และ StageScreen ตำแหน่งเดียวกัน
  4. กดจุดที่ผ่านมาแล้วเห็นสรุปถูกต้อง ตรงกับที่ ReportScreen ตอนจบเกมโชว์
  5. จุดอนาคตกดไม่ได้จริง
```
