## ปัญหาที่ต้องแก้ (ผ่านการวิเคราะห์ layout ปัจจุบันมาแล้ว)

1. หน้าจอเดิมเป็น carousel (ปุ่ม ◀▶ เลื่อนดูทีละสไตล์) แต่การเลือกสไตล์เป็นการตัดสินใจ
   "เปรียบเทียบ" 4 ตัวเลือก — carousel บังคับให้ผู้เล่นจำตัวเลขของสไตล์ก่อนหน้าไว้ในหัวเอง
2. ในการ์ดรายละเอียด ข้อมูล 6 ก้อน (persona, adjustLabel, ต้องตัดสินใจบ่อยแค่ไหน+FreqDots,
   pros/cons, stat pills, lesson) มีน้ำหนักภาพเท่ากันหมด (ฟอนต์ไซซ์เดียวกันทั้งบล็อก) ไม่มี
   จุดโฟกัสว่าอันไหนคือสิ่งที่ต้องดูก่อนเพื่อตัดสินใจ
3. สถิติ "ผลตอบแทน" (returnMult, บวก=ดี) กับ "แรงกระแทก" (shockMult, ลบ=ดี) มีขั้วตรงข้ามกัน
   ผู้เล่น ม.ปลาย ที่ไม่มีพื้นฐานต้องตีความสองชั้นถึงจะรู้ว่าตัวเลขติดลบคือเรื่องดี

## Layout ใหม่ที่ต้อง implement (validate กับผู้ใช้ผ่าน mockup แล้ว)

### 1. แถบเทียบ 4 สไตล์ (เพิ่มใหม่ อยู่บนสุด แทน/เสริมแถบ tab เล็กเดิมด้านล่าง)
Grid/row ของการ์ดเล็ก 4 ใบ แสดงพร้อมกันตลอดเวลา แต่ละใบมี:
- ไอคอน/portrait เล็ก (ภาพนิ่งเท่านั้น — ห้ามใส่ animation ตรงนี้ เพราะจะแย่งโฟกัสตอนเทียบตัวเลข)
- ชื่อสไตล์
- FreqDots (ความถี่ปรับพอร์ต)
- กำไรเฉลี่ย (ดูข้อ 3)
คลิกใบไหนก็ตั้ง index/selected style นั้น (ใช้ state เดิมที่มีอยู่แล้วสำหรับ carousel ได้เลย
ไม่ต้องสร้าง state ใหม่) ใบที่ถูกเลือกอยู่ให้มี border เด่นกว่าใบอื่น

### 2. การ์ดรายละเอียด (ของสไตล์ที่เลือกอยู่ 1 ใบ) — จัดลำดับใหม่จากบนลงล่าง
1. ชื่อสไตล์ + badge "แนะนำสำหรับเล่นครั้งแรก" (ถ้ามี) — เหมือนเดิม
2. **แถวสถิติหลัก (เพิ่มใหม่ อยู่บนสุดของเนื้อหา ก่อน persona/tagline ใดๆ)** — 2 กล่องสถิติขนาดใหญ่
   เด่นชัด (กำไรเฉลี่ย, ป้องกันความเสี่ยง) + กล่อง FreqDots เล็กกว่าด้านล่าง วางคู่กับโซน
   ภาพตัวละคร (portrait/GIF) ทางซ้าย:
   - "ปรับพอร์ตได้/ครั้ง" → FreqDots (component เดิม)
   - "กำไรเฉลี่ย" → +{Math.round((style.returnMult - 1) * 100)}%
   - "ป้องกันความเสี่ยง" → ค่าที่พลิกขั้วจาก shockMult ให้เป็น "ยิ่งมากยิ่งดี" เสมอ:
     Math.round((1 - style.shockMult) * 100) — ใส่เครื่องหมาย + ถ้าค่าเป็นบวก
     (ถ้าสไตล์ไหน shockMult > 1 ค่านี้จะติดลบได้จริง แปลว่าสไตล์นั้นรับแรงกระแทกหนักกว่าฐาน —
     แสดงเป็นค่าติดลบสีแดง/muted ตามปกติ อย่าซ่อนหรือบังคับให้เป็นบวกเสมอ)
   ทั้งสองสถิตินี้ต้องใช้กฎขั้วเดียวกัน (เลขบวก = ดีเสมอ) ไม่ใช้ป้ายเดิม "ผลตอบแทน"/"แรงกระแทก" อีก

   **เพิ่มคำอธิบายสั้น 1 บรรทัดใต้ตัวเลขแต่ละก้อน (ฟอนต์เล็กกว่าตัวเลข, `text-secondary`)**
   ผู้เล่น ม.ปลาย ดูตัวเลข % เปล่าๆ แล้วไม่เข้าใจว่ามันต่างจากสไตล์อื่นยังไงในทางปฏิบัติ —
   ให้แปลตัวเลขเป็นประโยคเทียบกับ "นักลงทุนระยะกลาง" ซึ่งเป็นค่ามาตรฐานทางคณิตศาสตร์อยู่แล้ว
   ในข้อมูลจริง (`medium.returnMult === 1.0 && medium.shockMult === 1.0`) ใช้สูตร:
   - ถ้า id === 'medium': โชว์ "ค่ามาตรฐานที่สไตล์อื่นเทียบด้วย" ทั้งสองช่อง (ไม่ใช่ "+0%" เฉยๆ
     เพราะ 0% อ่านแล้วดูไร้ค่า ทั้งที่จริงคือ "นี่คือหน่วยวัด")
   - สไตล์อื่น: `รวยกว่าสายกลาง {n}% ตอนจบเกม` ถ้า returnMult > 1, หรือ
     `จนกว่าสายกลาง {n}% ตอนจบเกม` ถ้า returnMult < 1 (เทรดเดอร์: ต่อท้ายด้วย
     "(ยังไม่รวมค่าธรรมเนียมที่กัดเพิ่ม)" เพราะ tradeFeePct ทำให้แย่กว่าตัวเลขนี้อีก)
   - ป้องกันความเสี่ยง: `เสียหายน้อยกว่าปกติ {n}% เวลาเจอวิกฤต` ถ้าดีกว่าฐาน หรือ
     `รับแรงกระแทกเท่าคนทั่วไป ไม่มีการป้องกันพิเศษ` ถ้าเท่าฐาน (shockMult === 1.0 พอดี
     ตอนนี้เป็นแบบนี้ทั้ง medium/trader/vi มีแค่ longterm ที่ต่างจากฐานจริง)

   **สำคัญ — อย่าซ่อนตัวเลขลบของเทรดเดอร์:** `styles.js` มี comment ยืนยันว่าเทรดเดอร์
   ถูกออกแบบให้เป็น "กับดักที่ตั้งใจให้ติด" (returnMult 0.85 บวกค่าธรรมเนียมสะสม = แพ้
   ระยะยาวเกือบตลอด) — คำอธิบายที่เพิ่มต้องบอกตรงๆ ว่าสไตล์นี้ทำให้จนกว่าค่ามาตรฐาน
   ไม่ใช่ปัดเป็นคำกลางๆ อย่าง "จุดอ่อน" ลอยๆ เพราะจุดประสงค์ของกับดักนี้คือให้ผู้เล่นรู้ล่วงหน้า
   แล้วยังเลือกเพราะอยากคุมพอร์ตบ่อยๆ พอแพ้จริงจะได้รู้สึกว่า "เกมบอกฉันแล้ว" ไม่ใช่ถูกหลอก
3. **โซนภาพตัวละคร (ซ้าย ควบคู่กับข้อ 2)** — เช็คว่า repo มี src/components/PixelSprite.jsx
   อยู่แล้วหรือไม่ (มาจากระบบต่อสู้เดิม) ถ้ามี logic sprite animation ที่ใช้ซ้ำได้ให้ใช้ต่อ
   แทนสร้างใหม่ ถ้าไม่เหมาะให้ใช้ Portrait component เดิมไปก่อน (ภาพนิ่ง) แล้วค่อยเพิ่ม
   animation ทีหลังได้ — เฉพาะภาพของสไตล์ที่เลือกอยู่เท่านั้นที่ควรขยับ ภาพในแถบเทียบข้อ 1
   ต้องนิ่ง
4. เส้นคั่น แล้วตามด้วยเนื้อหารอง (ฟอนต์เล็กลง ไม่แข่งกับสถิติหลักด้านบน) ตามลำดับเดิม:
   - กล่อง adjustLabel() (คำอธิบายกลไก เช่น "ตั้งพอร์ตต้นบทครั้งเดียว แล้วปล่อยยาวจนจบบท")
   - pros (✓ สีเขียว) / cons (✗ สีแดง)
   - lesson — ตัวเล็กสุด สีจางสุด อยู่ล่างสุด
