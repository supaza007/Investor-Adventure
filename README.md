# 📈 เส้นทางชีวิตนักลงทุน — เกมสอนการลงทุน

เกมจัดพอร์ตที่พาคุณเดินจากอายุ 20 ถึงเกษียณ 60 ใน 10-15 นาที
สร้างด้วย **React + Tailwind CSS v4 + Vite**

> คำกริยาหลักของเกมคือ **"จัดสรร"** ไม่ใช่ "โจมตี" — เงินเฟ้อไม่ใช่บอสให้ฆ่า
> มันคือสภาพอากาศที่ต้องอยู่กับมันให้ได้

## วิธีเปิดเล่น

**วิธีที่ง่ายที่สุด — ดับเบิลคลิก**
เปิดไฟล์ `dist/index.html` ด้วยการดับเบิลคลิกได้เลย (build เป็นไฟล์เดียวรวมทุกอย่าง ไม่ต้องมีเซิร์ฟเวอร์)
> ถ้าแก้โค้ดแล้ว ต้องรัน `npm run build` ใหม่ก่อน ไฟล์ `dist/index.html` ถึงจะอัปเดต

**สำหรับพัฒนา/แก้เกม (มี hot-reload)**
```bash
npm install     # ครั้งแรกครั้งเดียว
npm run dev
```
เปิดเบราว์เซอร์ที่ http://localhost:5173

## วิธีเล่น

1. **เลือกสไตล์นักลงทุน** — ทุกสไตล์ซื้อสินทรัพย์ได้เหมือนกันหมด ต่างกันที่ *แตะพอร์ตได้บ่อยแค่ไหน*
2. **จัดพอร์ต** ก่อนรู้ว่าทศวรรษนี้จะเกิดอะไรขึ้น (ตัดสินใจภายใต้ความไม่แน่นอน เหมือนของจริง)
3. **เหตุการณ์คลี่ตัว 5 สเตจ** — สัญญาณเตือน → เผยประเภท → แรงกระแทก → จุดตัดสินใจ → สรุป
4. ทำซ้ำ 4 บท (อายุ 20-29 / 30-39 / 40-49 / 50-59) แล้วดู **รายงานผลเกษียณ**

**ไม่มีเฉลยตายตัว** — ทุกเครื่องมือมีจุดตายจริง แม้แต่ตราสารหนี้ (แพ้เงินเฟ้อ) และเงินสด (แพ้เงินเฟ้อหนักสุด)
ตัดสินใจถูกแล้วยังแพ้ได้เพราะดวง — เกมจะบอกตรงๆ ว่าคุณเตรียมดีแค่ไหน แยกจากดวงคุณเป็นยังไง

## คำสั่ง

```bash
npm run dev             # เล่นแบบ hot-reload
npm run build           # build เป็นไฟล์เดียวใน dist/ (ดับเบิลคลิกเปิด / ทำ .exe)
npm run build:web       # build แยกไฟล์ใน dist-web/ (สำหรับขึ้นเว็บ โหลดเร็วกว่ามาก)
npm test                # เทสต์เอนจินเกม 48 ตัว (ใช้ node --test ไม่ต้องลงอะไรเพิ่ม)
npm run sim             # จำลอง 2,000 เกม เช็คว่าเกมยังสอนถูกอยู่ไหม
npm run optimize-images # บีบรูปในโฟลเดอร์ assets ให้เล็กลงสำหรับเว็บ
npm run electron:dev    # เปิดเป็นแอปเดสก์ท็อป
npm run electron:build  # build เป็น .exe (Windows, portable)
```

## เรื่องภาพประกอบ (สำคัญถ้าจะแก้โค้ดต่อ)

ไฟล์รูปใน `src/assets/characters/`, `src/assets/events/` และ `title-*` **ไม่ถูก commit ขึ้น git**
เพราะยังไม่ทราบเจ้าของลิขสิทธิ์ (ดูเหตุผลใน `.gitignore`)

โค้ดจึงไม่ import รูปตรงๆ แต่ใช้ `import.meta.glob` สแกนว่ามีไฟล์อะไรอยู่ตอน build:

| เครื่อง | ผลลัพธ์ |
|---|---|
| เครื่องที่มีไฟล์รูป (เครื่องผู้พัฒนา) | ใช้รูปจริง |
| เครื่องที่ไม่มี (GitHub Actions ตอน deploy) | ใช้ emoji + ชื่อเกมแบบตัวอักษร |

**โค้ดชุดเดียวใช้ได้ทั้งสองที่** ไม่ต้องแก้ไฟล์ก่อน deploy

เวลาเพิ่มรูปใหม่: วางไฟล์ชื่อ `<id>.webp` ให้ตรงกับ id ใน `src/game/engine/data/`
แล้วรัน `npm run optimize-images` — ไม่ต้องแก้โค้ดเลย
ถ้าวันไหนได้รูปที่มีสิทธิ์เผยแพร่ ให้ลบบล็อก "ภาพประกอบ" ใน `.gitignore` ทิ้งแล้ว commit ตามปกติ

## เอาขึ้นเว็บ

`.github/workflows/deploy.yml` ตั้งไว้ให้แล้ว — push ขึ้น branch `main` บน GitHub
แล้วมันจะรันเทสต์ → ตรวจบาลานซ์ → build → ปล่อยขึ้น GitHub Pages ให้อัตโนมัติ
(ครั้งแรกต้องไปเปิด Settings → Pages → Source = "GitHub Actions" ก่อน)

ถ้าเทสต์ไม่ผ่านหรือบาลานซ์เกมพัง มันจะไม่ปล่อยขึ้นเว็บ

### `npm run sim` คืออะไร

เกมเวอร์ชันก่อนหน้าพังเพราะไม่มีใครรู้ว่าผู้เล่นหมดตัวเลือกให้คิดตั้งแต่เทิร์นที่ 3 จนกระทั่งเอาไปให้คนเล่นจริง
`sim` เล่นเกมจบ 2,000 รอบด้วยกลยุทธ์ต่างๆ แล้วตอบว่า **"กระจายความเสี่ยงชนะจริงไหม"** และ
**"มีพอร์ตไหนเป็นเฉลยตายตัวไหม"** — ถ้าจูนตัวเลขใน `balance.js` แล้วเกมสอนผิด sim จะฟ้องทันที

## โครงสร้างโปรเจกต์

```
├─ docs/
│  ├─ plans/             ดีไซน์ + สเปกตัวเลขเอนจิน (อ่านก่อนแก้บาลานซ์)
│  └─ reference/         เนื้อหาความรู้ที่เก็บไว้อ้างอิง (ยังไม่ได้ใช้ในเกม)
├─ scripts/
│  ├─ sim.mjs            จำลองเกม 2,000 รอบ เช็คว่าเกมยังสอนถูก
│  └─ optimize-images.mjs  บีบรูปสำหรับเว็บ (GIF → WebP เคลื่อนไหว)
├─ assets-source/        ต้นฉบับภาพ/เสียงก่อนบีบ (ไม่ขึ้น repo)
├─ electron/             Electron main process สำหรับ build .exe
└─ src/
   ├─ App.jsx            สลับหน้าจอตาม phase + เสียงคลิกสังเคราะห์
   ├─ game/engine/       เอนจินเกม — JS ล้วน ไม่มี React ไม่มีการสุ่มที่ซ่อนอยู่
   │  ├─ balance.js      ⭐ ตัวเลขจูนได้ทุกตัวรวมที่เดียว (แก้ตรงนี้ที่เดียวพอ)
   │  ├─ gameState.js    state machine 4 บท × 5 สเตจ
   │  ├─ encounter.js    สูตร exposure / outcome band / margin call
   │  ├─ portfolio.js    คณิตศาสตร์พอร์ต (HHI, การกระจุกตัว, ทบต้น)
   │  ├─ scam.js         กลไกมิจฉาชีพ (ไม่ใช่แค่ tag matching)
   │  ├─ report.js       รายงานผลเกษียณ
   │  ├─ rng.js          ตัวสุ่มที่กำหนด seed ได้ (เกม replay ได้)
   │  ├─ data/           เครื่องมือ 6 ชนิด · เหตุการณ์ 12 ตัว · สไตล์ 4 แบบ
   │  └─ *.test.js       เทสต์ 48 ตัว
   ├─ assets/            ภาพที่เกมใช้จริง (ไม่ขึ้น repo — ดูหัวข้อด้านบน)
   └─ components/        UI 7 หน้าจอ + Portrait/ToolTheme/art
```

หน้าจอทั้งหมดมี 5 หน้า เดินตามลำดับนี้:
`CoverScreen` → `StyleSelect` → `AllocationScreen` ⇄ `StageScreen` → `ReportScreen`

## จะจูนบาลานซ์ยังไง

แก้ [`src/game/engine/balance.js`](src/game/engine/balance.js) ที่เดียว แล้วรัน:

```bash
npm run sim && npm test
```

ถ้าเกมเริ่มสอนผิด (เช่น มีพอร์ตที่ชนะขาดจนไม่ต้องคิด) sim จะขึ้น ❌ พร้อมบอกว่าพังข้อไหน
ตัวเลขความอ่อนไหวของสินทรัพย์แต่ละชนิดอยู่ใน [`data/tools.js`](src/game/engine/data/tools.js)
